Introduction

BlackRoad OS is a distributed network of autonomous agents coordinated via NATS event bus and K3s orchestration. Each agent maintains persistent memory, processes task streams, and communicates through structured events.

What is BlackRoad OS

BlackRoad OS provides a runtime for spawning, managing, and coordinating AI agents at scale. Agents are first-class primitives with identity, memory, and task-processing capabilities built in.

Note
BlackRoad OS requires Node.js 20+ and a running NATS server. K3s is optional for single-node deployments.

Core Architecture

The system is built on three pillars: agents (autonomous workers), streams (event transport via NATS), and memory (persistent context per agent).

// Spawn an agent with memory and stream bindings
const agent = await Agent.spawn({
  id: 'cecilia',
  role: 'orchestrator',
  memory: { max_mb: 512, persist: true },
  streams: ['tasks.inbound', 'events.system']
})

// Agent is now active and processing
console.log(agent.status) // → 'active'

Quick Start

1
Install the CLI
npm install -g @blackroad/cli
2
Initialize a project
br init my-project
3
Spawn your first agent
br agent spawn --id cecilia --role orchestrator
4
Send a task
br task create --agent cecilia --payload '{"action": "hello"}'
Important
Ensure your NATS server is running before spawning agents. Agents will fail to connect without an active event bus.
